# 📋 Шпаргалка для конференции

## Структура презентации

### 1. Введение (2 мин)
- Тема: "Музыкальный плейлист как отражение личности"
- Цель: анализ личности через музыкальные предпочтения
- Инструмент: реляционная база данных PostgreSQL

### 2. Архитектура БД (3 мин)

**4 таблицы:**
- `users` — пользователи (имя, возраст, пол)
- `tracks` — треки с характеристиками (energy, valence)
- `playlists` — плейлисты пользователей
- `playlist_tracks` — связь треков и плейлистов (многие-ко-многим)

**Ключевые поля треков:**
- `energy` (0-1) — интенсивность, громкость, темп
- `valence` (0-1) — позитивность, настроение

### 3. Источник данных (2 мин)
- Характеристики треков через OpenRouter API (Llama 3)
- Альтернатива: Spotify Audio Features API
- 8 участников, 160 треков в БД

### 4. Алгоритм анализа (3 мин)

**4 зоны треков:**
1. Hype (energy ≥ 0.6, valence ≥ 0.6) — энергия + позитив
2. Aggressive (energy ≥ 0.6, valence < 0.45) — энергия + негатив
3. Sad (energy < 0.5, valence < 0.45) — тихо + грустно
4. Calm (energy < 0.5, valence ≥ 0.6) — тихо + позитивно

**Логика:**
- Считаем долю треков в каждой зоне
- Если зона > 30% → доминирующий тип
- Иначе → смешанный тип

### 5. Типы личности (2 мин)

1. **Эйфорический экстраверт** — энергичный позитив
2. **Тревожный активист** — энергия + тревога
3. **Меланхолик** — тихая грусть
4. **Созерцательный оптимист** — спокойный позитив
5. **Меланхолик с агрессией** — смесь грусти и злости
6. **Позитивный микс** — разнообразный позитив
7. **Эмоционально нестабильный** — нет доминанты

## SQL-запрос (для показа)

```sql
SELECT
    u.name,
    ROUND(COUNT(*) FILTER (WHERE t.energy >= 0.6 AND t.valence >= 0.6) * 1.0 / COUNT(*), 2) AS hype_ratio,
    CASE
        WHEN COUNT(*) FILTER (WHERE t.energy >= 0.6 AND t.valence >= 0.6) > COUNT(*) * 0.30
            THEN 'Эйфорический экстраверт'
        WHEN COUNT(*) FILTER (WHERE t.energy >= 0.6 AND t.valence < 0.45) > COUNT(*) * 0.30
            THEN 'Тревожный активист'
        -- ... остальные условия
        ELSE 'Эмоционально нестабильный'
    END AS personality_type
FROM users u
JOIN playlists pl ON pl.user_id = u.id
JOIN playlist_tracks pt ON pt.playlist_id = pl.id
JOIN tracks t ON t.id = pt.track_id
GROUP BY u.name;
```
